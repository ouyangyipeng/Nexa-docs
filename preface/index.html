    1. 前言：重构智能体开发范式¶

    在深入 Nexa 的核心语法之前，让我们退一步思考：当前业界在开发以大语言模型（LLM）为核心的智能体（Agent）系统时，究竟面临着怎样的结构性困境？Nexa 是如何基于编译器理论与语言设计学脱颖而出，实现从“应用层框架”到“底层原语”的降维打击的？

    本前言部分被划分为两个核心深度长文，我们强烈建议您在了解核心语法前阅读它们，以理解 Nexa 的设计初衷：

    • 📖 调研背景：从语言模型到 Agent 原生计算范式：全面梳理大语言模型与 Agent 的演进，深度剖析 Dify / Coze（弱代码范式）与 Python / LangChain（通用编程语言范式）所面临的底层工程痛点，并阐述 Nexa 如何作为“范式三”重塑原生原语。
    • ⚔️ 典型场景：Nexa 与现有编排方法的深度对比：通过复杂代码生成、长周期金融研报生成以及需要“人在回路（HITL）”的高风险 IT 运维等三大典型场景，实战化对比 Nexa 与其他框架的降维打击优势。
